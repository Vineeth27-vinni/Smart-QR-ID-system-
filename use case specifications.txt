Description: Student can take admission or enroll for current semester

Pre-Condition: valid seat allocation details for new admission. Promoted, valid student for current semester registration

Post-Condition: registration acknowledgment and generate QR's  to access library and academics

Updated By: -
Updated Date: -
Actor: Student

Description: Students can view hostel details and mess details. Inside the hostel/mess module, students can access mess, book meals, view bills, pay bills and raise complaints.

Pre-Condition: Student must have valid login credentials and should be a resident of that particular hostel.

Post-Condition: Student can access hostel and mess services such as booking meals, viewing bills, paying bills and raising complaints.

Assumptions: Hostel opt-in data is available in the system. Mess billing system is functional. Payment system is available. Complaint system is operational.

Use Case ID: 009
Name: Librarian
Created By: Vineeth and Ram
Created Date: 26-02
Updated By: -
Updated Date: -
Actor: Librarian

Description: Librarian manages library operations including verifying students through QR scan, issuing books, accepting returns, and maintaining library records.

Pre-Condition: Librarian must have valid login credentials and student must have completed semester registration

Post-Condition: Books are issued or returned successfully and library records are updated

Normal Flow: Librarian logs into the system. Librarian scans student QR code. System verifies student eligibility. Librarian checks book availability. Librarian issues the book to the student. Librarian records book return when returned. System updates borrowing records. Librarian logs out

Alternate Flow: If login credentials are invalid, access is denied. If student is not eligible, book cannot be issued. If book is not available, issue request is denied

Frequency: Frequent

Business Rule: Books can be issued only to eligible students with valid registration and within borrowing limits

Assumptions: Library database is updated. Student eligibility status is available. QR verification system is functioning properly
